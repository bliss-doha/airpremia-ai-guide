01
TERMS · 용어 정의

먼저 알아두면 좋은 6가지

가이드를 읽기 전, 아래 6가지 개념을 먼저 알아두면 도움이 됩니다.

생성형 AI
사람의 질문에 글·이미지·음성을 만들어주는 AI입니다.

예시: ChatGPT, Claude, Gemini, Copilot
Agentic AI
답변만 하는 것을 넘어 사용자를 대신해 실제 작업을 수행하는 AI입니다. 메일 발송, 일정 생성, 파일 정리 같은 행동(Action)을 직접 실행합니다.

예시: "내일 회의 일정 잡고 참석자에게 메일 보내줘"
기업용 AI
회사가 사업자와 직접 계약해 임직원에게 제공하는 AI입니다. 입력 데이터는 학습에 사용되지 않습니다 (계약상 보장).

예시: 회사 라이선스 M365 Copilot, Claude Team
상용 AI
임직원이 개인 계정으로 가입한 AI입니다. 유료 여부와 무관하며, 기본값이 학습 ON인 경우가 많습니다.

예시: ChatGPT Plus, Gemini Pro, Claude Max
학습 (Training)
AI 모델을 더 똑똑하게 만들기 위해 데이터를 활용하는 과정입니다.

내 입력이 학습에 쓰이면 다른 사용자 답변에 내 정보가 섞여 나올 수 있습니다.
옵트아웃 (Opt-out)
"내 데이터는 학습에 사용하지 마세요"라고 설정하는 것입니다.

예시: ChatGPT 설정의 "모두를 위한 모델 개선" OFF
마스킹
이름·전화번호 같은 개인정보를 다른 표현으로 가리는 것입니다.

예시:
· 홍길동 → 고객 A
· 010-1234-5678 → 010-****-****
MFA (2단계 인증)
비밀번호 외에 휴대폰 인증번호 등을 추가로 입력하는 방식입니다. 계정이 노출되어도 사고 예방이 가능합니다.

참고: 에어프레미아는 OKTA를 사용합니다
02
CLASSIFICATION · 분류

기업용 AI vs 상용 AI

AI 서비스 제공 방식은 두 가지로 분류됩니다.

Enterprise · 회사 계약

기업용 AI

  • 계약 주체회사가 사업자와 직접 계약 (위수탁 체결)
  • AI 학습입력 데이터는 모델 학습에 사용되지 않음 (계약 및 서비스 정책 기준)
  • 로그인기업 SSO 지원 (OKTA 등)
  • 회사 데이터일반·기밀 작업 가능 (단 고객 개인정보 입력 금지)
  • 관리·감사사용자·접근 권한·감사 로그 관리 가능
Consumer · 개인 계정

상용 AI

  • 계약 주체임직원 본인이 가입·결제한 미승인 개인 계정
  • AI 학습서비스 정책에 따라 입력 데이터가 모델 개선에 활용될 수 있음
  • 로그인개인 이메일
  • 회사 데이터회사 기밀·개인정보 입력 제한 (공개 자료 작업에 한정)
  • 관리·감사사업자 정책에 종속, 회사 통제 어려움
에어프레미아 공식

회사가 정식 도입한 AI 서비스

회사 업무용 AI는 다음과 같으며, airpremia.com 도메인을 가집니다.

Microsoft 365 Copilot
MICROSOFT · ENTERPRISE LICENSE
  • 회사 M365 라이선스 기반 제공
  • Copilot Chat 사용 가능
  • M365 내 자원 검색 가능
Claude Team
ANTHROPIC · TEAM LICENSE
  • 회사 워크스페이스 단위 제공
  • Chat: 기본 질의·문서 작성
  • Cowork: 업무 자동화 지원
  • Code: 개발 및 스크립트 작성
03
RULES · 이용 수칙

AI 이용 전 5가지 수칙

AI 사용 시 아래 5가지 수칙은 꼭 지켜주세요.

RULE 01

회사 자료는 회사 공식 AI에서 다룹니다

회사가 도입한 공식 AI(M365 Copilot, Claude Team)에서 회사 자료를 다룹니다. 상용 AI에는 회사 자료를 올리지 않습니다.

RULE 02

인증정보는 어떤 AI에도 입력하지 않습니다

비밀번호·API Key·Token은 어떤 AI에도 입력하지 않습니다. 인증정보는 Vault 등 별도 보관소를 사용합니다.

민감정보 관련 문의는 Slack #help_정보보안으로 부탁드립니다.

RULE 03

민감한 정보는 먼저 가립니다

개인정보·고객정보·계약정보는 그대로 입력하지 않습니다. 필요한 경우 이름·번호 등을 다른 표현으로 바꿔 사용합니다.

예시:
· 홍길동 → 고객 A
· 전화번호 → 010-****-****

RULE 04

상용 AI는 학습 옵션을 끄고 사용합니다

상용 AI는 기본값이 학습 ON인 경우가 많습니다. 설정 가이드에 따라 학습 옵션을 OFF로 변경한 뒤 사용합니다.

RULE 05

AI 결과물은 사람이 검토한 뒤 사용합니다

AI는 그럴듯하지만 틀린 답(환각)을 만들 수 있습니다. 코드·문서·분석 결과를 외부에 공유하기 전에는 사람이 한 번 검토합니다.

고객 안내·법률 문서·보안 관련 작업은 반드시 담당자가 최종 확인합니다. AI 결과물의 최종 책임은 사용자에게 있습니다.

DATA GRADES · 데이터 등급

4단계 신호등으로 판단합니다

입력하려는 데이터가 어떤 등급인지 확인하고, 등급에 맞는 AI에서 다룹니다.

공개 자료
  • 홈페이지·보도자료·공식 SNS 등 외부에 이미 공개된 정보
기업용 · 상용 AI 사용 가능
사내 일반
  • 내부 회의·공지·비식별화 자료
  • 민감도 낮은 업무 자료
기업용 · 상용 AI 사용 가능
(AI 학습 비활성화 필수)
기밀 · 사내 개인정보
  • 미공개 문서·계약·재무·인사 정보
  • 임직원 개인정보, 고객 PNR
기업용 AI만 사용 가능
인증정보 · 고객 식별정보

  • 고객 이름·여권·연락처 등
모든 AI 입력 금지

판단 4단계 흐름

위에서 아래로 순서대로 확인합니다. 위험도가 높은 항목부터 체크하여, 해당하는 단계에서 멈추고 그 조치를 따릅니다.

1
인증정보 또는 고객 식별정보가 포함되어 있나요? (비밀번호·API Key·Token·Secret 등, 고객 이름·여권·연락처 등)
YES모든 AI 입력 금지. 노출되었다면 즉시 폐기·재발급
NO2단계로
2
회사 기밀 또는 사내 개인정보인가요? (미공개 문서·계약·재무·인사, 임직원 개인정보, 고객 PNR)
YES기업용 AI(M365 Copilot, Claude Team)에서만 사용
NO3단계로
3
외부에 공개되지 않은 사내 자료인가요? (내부 회의·공지·비식별화된 자료)
YES기업용 · 상용 AI 가능 (상용은 학습 OFF 필수)
NO4단계로
4
외부에 이미 공개된 자료인가요? (홈페이지·보도자료·공식 SNS)
YES기업용 · 상용 AI 모두 사용 가능
💬 민감정보 관련 문의 : 등급 판단이 모호하거나 처리 방법이 확실하지 않은 경우, Slack #help_정보보안 채널로 문의 부탁드립니다.
05
SCENARIOS · 실전 시나리오

좋은 예 vs 나쁜 예

실제 업무 상황에서 어떻게 행동하면 좋은지 사례로 살펴봅니다.

시나리오 01 · 고객 응대 메일 초안 작성
나쁜 예
"홍길동 고객(010-1234-5678, 예약번호 AB123456)이 환불 요청한 사례 정리해줘"
→ 개인정보가 외부로 전송되어 위수탁 미체결로 법적 문제가 발생할 수 있습니다.
좋은 예
"환불 요청 고객 대응 메일을 정중한 톤으로 초안 작성해줘. {이름}, {예약번호}를 포함한 일반적 사례 기준으로."
→ 실제 식별정보 대신 플레이스홀더({이름}·{예약번호})로 일반 양식만 요청합니다.
시나리오 02 · 운항 매뉴얼 요약
나쁜 예
운항 매뉴얼 PDF를 개인 Gemini에 업로드해 요약 요청
→ 기밀자료가 외부로 전송됩니다. 학습에 활용되어 다른 사용자 답변에 섞일 가능성이 있습니다.
좋은 예
M365 Copilot에서 매뉴얼 SharePoint 파일을 참조해 요약 작성
→ 회사 테넌트 내부에서 처리됩니다. 외부 전송이 없습니다.
시나리오 03 · 코드 디버깅 도움 받기
나쁜 예
API Key, DB 접속 정보가 들어있는 코드 그대로 개인 ChatGPT에 붙여넣기
→ 인증정보가 노출됩니다. 노출된 정보는 즉시 폐기·재발급 수행합니다.
좋은 예
API Key 부분은 "[REDACTED]"로 가린 채 Claude Team에 입력해 로직 디버깅 요청
→ 로직만 검토합니다. 인증정보는 입력하지 않습니다.
06
AI 자동화 · 안전 사용 가이드

AI가 직접 작업할 때는 한 번 더 확인합니다

AI는 실제 작업도 수행할 수 있습니다. 실행 전 한 번 더 확인합니다.

AI 자동화란?

AI가 사용자를 대신해 메일 발송·일정 등록·파일 정리 같은 작업을 수행하거나, 여러 서비스를 연결해 작업을 자동으로 처리하는 기능입니다. 편리하지만 잘못된 작업도 실행될 수 있습니다.

01

사람이 검증합니다

AI 자동화는 바로 운영에 적용하지 않고, 테스트를 통해 정상 동작 여부를 먼저 확인합니다. 적용 후에도 정상 동작 여부를 지속적으로 확인합니다.

02

작업은 더 신중하게

발송·결제·삭제·서명은 한 번 실행되면 되돌리기 어렵습니다. 이런 작업은 사람이 최종 승인 후 실행합니다.

03

권한은 필요한 만큼만 줍니다

AI에게 메일·캘린더·파일 접근 권한을 줄 때 읽기·쓰기·삭제 중 꼭 필요한 것만 허용합니다. 사용 후에는 권한을 제거합니다.

사용 예시

권장

회의록 자동 작성: AI가 회의 녹취를 요약해 회의록 초안을 만들고, 사람이 검토한 뒤 참석자에게 발송합니다.

주의

고객 응대 자동 회신: AI가 고객 문의에 자동 회신하는 경우, 정확성·톤·법적 표현을 사람이 마지막에 확인합니다.

권장

일정 후보 제안: AI가 캘린더를 확인해 가능한 시간대 후보를 제안하고, 사람이 최종 선택해 등록합니다.

주의

파일 정리·삭제 위임: AI에게 "오래된 파일 정리"시 1차적으로 정리 목록을 받고 삭제를 승인합니다.

07
CONNECTORS · 외부 서비스 연결

AI에 다른 서비스를 연결할 때 (MCP)

AI 연결 기능은 메일·문서·파일에 접근 권한을 가질 수 있습니다.

01

승인된 기업용 AI에서만 연결

M365·Slack·GitHub·Notion 등 회사 자산은 승인된 기업용 AI에서만 연결합니다.
개인 계정 기반 AI에는 연결하지 않습니다.

02

개인 클라우드도 주의

개인 Google Drive·Dropbox에도 업무 자료가 포함될 수 있습니다.
상용 AI에 연결 시 업무 정보가 함께 노출될 수 있습니다.

03

사용하지 않는 연결은 해제

한 번 연결한 서비스는 연결이 계속 유지됩니다.
분기마다 연결 목록을 확인하고 사용하지 않는 연결은 해제합니다.

⚠ 기업용 AI만 회사 자산 연동 가능

회사 자산 연동은 승인된 기업용 AI에서만 사용합니다. 개인 계정 기반 AI에는 연결하지 않습니다.
💡 왜 주의해야 하나요?

연결된 서비스는 AI가 지속적으로 접근할 수 있습니다. 예상하지 못한 작업이 수행될 수 있으므로 필요한 서비스만 연결합니다.

CLAUDE SECURITY · Claude 보안 가이드

Claude 보안 가이드

Claude 사용 시 지켜야 할 보안 수칙을 한눈에 정리했습니다.

Claude Team
ANTHROPIC · TEAM LICENSE
TEAM
환경설정
  • 01
    로그인 확인: 회사 워크스페이스로 접속했는지 매번 확인합니다. 개인 계정과 섞이지 않도록 주의합니다.
  • 02
    프로필 최소화: 이름 외 불필요한 개인정보는 입력하지 않습니다.
  • 03
    메모리 기능 점검: 설정 → 기능 → 메모리에서 자동 기억 저장 여부를 확인합니다. 민감 대화가 저장되지 않도록 필요 시 끕니다.
  • 04
    커넥터 사용: Google Drive·GitHub·Slack 등 외부 연동은 IT 사전 승인 후 사용합니다 (섹션 07 참조).
  • 05
    데이터 정책 확인: Claude Team 입력 데이터는 모델 학습에 사용되지 않습니다 (계약 보장).
기능 사용
  • 01
    Project로 격리: 민감자료는 Project 안에서 작업합니다. 공유 범위를 매번 확인합니다.
  • 02
    코드 검토: AI가 만든 코드는 실행 전 사람이 검토합니다. 코드에 비밀번호·API Key·고객정보를 넣지 않습니다.
  • 03
    자동 실행 금지: 메일 발송·일정 등록·파일 수정 같은 작업은 자동으로 실행하지 않습니다. 검토 후 수동으로 진행합니다 (섹션 06 참조).
  • 04
    응답 검토: 외부로 내보내거나 시스템에 반영하기 전 사람이 한 번 확인합니다. 고객 안내·법률·보안 작업은 담당자 확인 후 사용합니다.
  • 05
    공유 전 점검: 대화를 외부와 공유하기 전 민감정보가 포함되어 있는지 확인합니다.
정보보안 주의사항
  • 01
    민감정보 입력 금지: 비밀번호·API Key·주민번호·고객정보·내부 영업비밀은 입력하지 않습니다.
  • 02
    파일 업로드 점검: 업로드 전 파일에 개인정보나 기밀이 포함되어 있는지 확인합니다.
  • 03
    응답 사실 확인: 숫자·법령·인용·링크는 원본 출처에서 다시 확인합니다. AI 응답은 틀릴 수 있습니다.
  • 04
    외부 문서 분석 시 주의: 웹페이지·이메일·문서에 숨겨진 지시문이 Claude의 동작을 바꿀 수 있습니다. 응답이 이상하면 즉시 중단합니다.
  • 05
    등급별 사용: 기밀·개인정보를 다루는 작업은 사용 전 정보보안실과 상의합니다.
  • 06
    이상 시 신고: 의심스러운 응답·계정 도용 의심·정보 유출 우려가 있으면 Slack #help_정보보안으로 즉시 신고합니다.

CONSUMER SETUP · 개인 AI 최소 안전 설정

개인 AI Opt-out 설정 가이드

개인 계정으로 상용 AI를 사용할 때 적용해야 할 최소 안전 설정입니다. 공개된 정보 작업에 한정해 사용합니다.

공통 원칙 3가지

① 회사명·직함·조직 정보를 프로필에 작성하지 않습니다
② 회사 계정 및 회사 자산(M365·Slack·GitHub 등)은 연결하지 않습니다
③ AI 학습 차단(Opt-out) 설정 후 사용합니다

LLM별 Opt-out 설정 방법

ChatGPT

OPENAI · Opt-out 설정
Settings → Data Controls
  1. "Improve the model for everyone" OFF (필수)
  2. Chat History & Training OFF (필수)
Settings → Personalization
  1. Reference Saved Memories OFF (권장)
  2. Reference Chat History OFF (권장)

※ Temporary Chat 사용 시 기록·학습 모두 OFF로 처리됩니다.

Claude

ANTHROPIC · Opt-out 설정
Settings → Privacy
  1. "Help improve Claude" OFF (필수)
  2. Location Data OFF (필수)
Settings → Features
  1. Search and reference chats OFF (권장)
  2. Generate memory from chat history OFF (권장)
  3. Network access OFF (권장)

※ 회사 Claude Team과 개인 Claude는 별도 워크스페이스입니다. 혼동에 주의하세요.

Gemini

GOOGLE · Opt-out 설정
Settings → Activity
  1. Gemini Apps Activity OFF (필수)
Settings → Personal Intelligence
  1. Memory OFF (권장)

※ 회사 Google Workspace 계정으로는 사용하지 않습니다.
※ 일부 UI에서는 "Keep Activity"로 표시될 수 있습니다.

Copilot

MICROSOFT · Opt-out 설정
Settings → Personal Settings
  1. Saved Memory OFF (필수)
  2. Chat History (Frontier) OFF (필수)
  3. Custom Instructions OFF (권고)

※ 회사 Microsoft 계정으로는 사용하지 않습니다. 회사 업무는 M365 Copilot에서 진행합니다.

CHECKLIST · 자가점검

5분 자가점검 체크리스트

새 AI 서비스를 처음 사용하거나, 새 업무에 AI를 도입하기 전 다음 항목을 확인합니다.

계정 · 인증
  • 사용하려는 AI가 기업용인지 상용인지 구분했습니다
  • 회사 계정(OKTA SSO)으로 로그인했습니다 (기업용 AI인 경우)
  • 2단계 인증(MFA)이 설정되어 있습니다
설정
  • 상용 AI는 학습 차단 설정을 적용했습니다
  • 외부 클라우드 연동은 필요한 경우만 켰습니다
  • 회사 자산 커넥터는 IT 승인 후 연결했습니다
입력 전 점검
  • 개인정보(이름·전화·주민번호·여권번호 등) 포함 여부 확인
  • 회사 기밀(미공개 노선·계약서·운항 매뉴얼) 포함 여부 확인
  • API Key·비밀번호·Token은 입력하지 않았습니다
자동화 · 파일 · 결과물
  • AI 자동 실행 작업은 사람이 검토 후 승인했습니다
  • 파일의 숨은 시트·주석·메타데이터를 확인했습니다
  • 화면 캡처 시 다른 창이 함께 찍히지 않았습니다
  • AI 결과물은 사람이 검토(HITL) 후 사용했습니다

FAQ · 자주 묻는 질문

자주 묻는 질문 10가지

임직원이 가장 자주 묻는 질문을 정리했습니다.

Q01내가 입력한 내용이 정말 AI 학습에 쓰이나요?
서비스와 설정에 따라 다릅니다. 상용 AI는 기본값이 학습 ON인 경우가 많아 직접 꺼야 합니다. 회사 계약 기업용 AI는 계약상 학습에 사용되지 않습니다.
Q02대화에서 '삭제' 버튼을 누르면 완전히 지워지나요?
화면에서만 사라지는 경우가 많습니다. 운영·보안·법적 의무 때문에 서버에 일정 기간 보관될 수 있고, 삭제 전에 이미 학습에 활용되었을 가능성도 있습니다. 처음부터 민감정보를 입력하지 않는 것이 가장 안전합니다.
Q03개인 ChatGPT Plus(유료)를 회사 업무에 써도 되나요?
개인 결제(ChatGPT Plus 등)와 회사 계약형 AI는 보안·데이터 정책이 다릅니다. 유료 여부와 무관하게 회사 계약이 아니면 상용 AI로 보며, 공개 자료 작업 정도만 가능합니다.
Q04한국에서 AI를 쓰는데 데이터가 해외로 갈 수도 있나요?
입력 데이터가 해외 리전에 저장·처리될 수 있습니다. ChatGPT/Claude/Gemini 모두 글로벌 서비스라 주요 데이터센터가 해외에 있습니다. 민감자료는 입력 전 회사 정책을 확인합니다.
Q05챗봇에 그냥 대화한 것도 개인정보가 되나요?
될 수 있습니다. "OO초 3학년 2반 담임", "지난주 OO병원 진료"처럼 여러 단서가 결합되면 특정 개인이 식별될 수 있습니다. 본인뿐 아니라 가족·동료·고객 정보 입력에도 주의합니다.
Q06회사 자료 요약·번역만 잠깐 써도 문제인가요?
입력 순간 외부 서비스로 전송될 수 있으므로 회사 정책 기준으로 판단합니다. 회사 비공개 자료는 회사 계약 AI(M365 Copilot, Claude Team)에서 작업합니다.
Q07AI에 Gmail·Drive를 연결하면 왜 위험할 수 있나요?
연결된 범위의 메일·문서·파일에 AI가 접근할 수 있습니다. 회사 계정은 연결하지 않고, 개인 계정도 사용 후 연동 해제를 권장합니다.
Q08AI가 만든 코드·문서를 그대로 써도 되나요?
사람이 한 번 검토(HITL)한 뒤 사용합니다. AI는 그럴듯하게 틀린 답을 만들기도 합니다(환각). 특히 보안 코드·고객 안내·법적 문서는 반드시 담당자 검토가 필요합니다.
Q09회의 녹음 파일을 AI에 올려도 되나요?
참석자 음성(개인정보)과 회의 내용(기밀)이 포함될 수 있습니다. 회사 계약 AI에서, 참석자 동의를 얻은 경우에 한해 사용합니다.
Q10실수로 개인정보를 입력했어요. 어떻게 하나요?
① AI 대화를 즉시 삭제합니다.
② 상용 AI 이용 시 본인이 직접 해당 AI 서비스에 데이터 삭제를 요청합니다 (각 서비스 Privacy Center 또는 고객지원). 외부 AI에 저장된 데이터는 사업자만 삭제할 수 있어 본인 요청이 가장 빠릅니다.
