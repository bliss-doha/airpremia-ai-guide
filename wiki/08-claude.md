# 08. Claude 보안 가이드

Claude 사용 시 지켜야 할 보안 수칙을 한눈에 정리했습니다.

## Claude Team
*ANTHROPIC · TEAM LICENSE*

### 환경설정
1. **로그인 확인**: 회사 워크스페이스로 접속했는지 매번 확인합니다. 개인 계정과 섞이지 않도록 주의합니다.
2. **프로필 최소화**: 이름 외 불필요한 개인정보는 입력하지 않습니다.
3. **메모리 기능 점검**: 설정 → 기능 → 메모리에서 자동 기억 저장 여부를 확인합니다. 민감 대화가 저장되지 않도록 필요 시 끕니다.
4. **커넥터 사용**: Google Drive·GitHub·Slack 등 외부 연동은 IT 사전 승인 후 사용합니다 (섹션 07 참조).
5. **데이터 정책 확인**: Claude Team 입력 데이터는 모델 학습에 사용되지 않습니다 (계약 보장).

### 기능 사용
1. **Project로 격리**: 민감자료는 Project 안에서 작업합니다. 공유 범위를 매번 확인합니다.
2. **코드 검토**: AI가 만든 코드는 실행 전 사람이 검토합니다. 코드에 비밀번호·API Key·고객정보를 넣지 않습니다.
3. **자동 실행 금지**: 메일 발송·일정 등록·파일 수정 같은 작업은 자동으로 실행하지 않습니다. 검토 후 수동으로 진행합니다 (섹션 06 참조).
4. **응답 검토**: 외부로 내보내거나 시스템에 반영하기 전 사람이 한 번 확인합니다. 고객 안내·법률·보안 작업은 담당자 확인 후 사용합니다.
5. **공유 전 점검**: 대화를 외부와 공유하기 전 민감정보가 포함되어 있는지 확인합니다.

### 정보보안 주의사항
1. **민감정보 입력 금지**: 비밀번호·API Key·주민번호·고객정보·내부 영업비밀은 입력하지 않습니다.
2. **파일 업로드 점검**: 업로드 전 파일에 개인정보나 기밀이 포함되어 있는지 확인합니다.
3. **응답 사실 확인**: 숫자·법령·인용·링크는 원본 출처에서 다시 확인합니다. AI 응답은 틀릴 수 있습니다.
4. **외부 문서 분석 시 주의**: 웹페이지·이메일·문서에 숨겨진 지시문이 Claude의 동작을 바꿀 수 있습니다. 응답이 이상하면 즉시 중단합니다.
5. **등급별 사용**: 기밀·개인정보를 다루는 작업은 사용 전 정보보안실과 상의합니다.
6. **이상 시 신고**: 의심스러운 응답·계정 도용 의심·정보 유출 우려가 있으면 Slack **#help_정보보안**으로 즉시 신고합니다.
