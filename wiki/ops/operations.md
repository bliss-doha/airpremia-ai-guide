# 운영 방침

## 1. 문서 소유
정보보안팀이 이 가이드를 소유합니다. 담당자·승인권자는 사내 문서 관리 기준에 따라 별도 관리합니다.

## 2. 개정 절차
1. 제안 — Slack `#help_정보보안` 또는 이슈 등록
2. 초안 작성 — 브랜치에서 수정
3. 보안팀 검토
4. `main` 브랜치 병합 = 자동 게시 (GitHub Pages 재빌드)

## 3. 버전 규칙
`vMAJOR.MINOR` 형식을 사용합니다.
- **MAJOR**: 정책 변경
- **MINOR**: 표현·오탈자 수정

이력은 [변경 이력](./changelog.html)으로 일원화합니다.

## 4. 개정 주기
- 반기 정기 검토
- 수시 검토 (신규 AI 도입·보안 사고·정책 변경 시)

## 5. 페이지 추가 규칙
- 파일명: `NN-slug.md`
- Front matter 필수 필드: `title`, `nav_order` (하위 페이지는 `parent`)
- 콜아웃 사용: 참고 `{: .note }` · 주의 `{: .warning }` · 금지 `{: .danger }` · 팁 `{: .tip }`
- 검색이 잘 되도록 핵심 용어를 제목·소제목에 포함합니다

## 6. 배포 반영
`main` 브랜치에 push하면 GitHub Pages가 자동으로 빌드합니다 (약 1~3분 소요). 빌드가 실패하면 직전 성공 빌드가 계속 서빙되므로 배포본은 깨지지 않습니다.
